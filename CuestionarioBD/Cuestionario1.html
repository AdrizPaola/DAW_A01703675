Preguntas

  • ¿Qué es entonces una base de datos - DB?
    Es un conjunto de información interelacionada organizada y estructurada con el fin de poder ser accesada, gestionada y actualizada de una manera rápida y sencilla para el análisis y/o la toma de decisiones en base a las necesidades de la empresa. Para el tratamiento de los datos de una base de datos (descripción de entidades y sus relaciones) existen algunos modelos como lo son: jerárquico, en red y relacional.[1]

  • ¿En qué casos es conveniente usar bases de datos?
    Se debe de considerar su uso cuando se trabaja con grandes cantidades de información y se desea mantener un control y un rápido acceso a ella. Su uso se ha extendido a diversos campos en la vida como lo pueden ser en hospitales para la catalogación de medicamentos y pacientes, en la escuela para matricular a los alumnos, en el comercio para controlar entradas y salidas de productos, etc. Siempre que se piense en una base de datos lo que se tiene en mente es lograr una exitosa compartición de datos, disminución de la redundancia como la duplicación de datos, restricciones de seguridad e integridad. [2]

  • ¿Qué es un sistema de gestión de base de datos - DBMS?
    Es un conjunto de software que nos ayuda a los usuarios al servir como una interfaz entre la base datos y las aplicaciones utilizadas, con el objetivo de manejar la información para su organización. Todo DBMS facilita la contrucción y manipulación de bases de datos a través de programas que trabajan de forma interrelacionada.[3]

  • Enlista y explica las funciones/responsabilidades que tienen los DBMS.
    • Abstracción de datos: Omitir detalles al usuario sobre el almacenamiento de los datos
    • Independencia: Modificación de esquemas de la base de datos sin afectar aplicaciones dependientes de ella.
    • Consistencia: Actualización de datos repetidos de manera simultánea
    • Seguridad: Garantizar que los datos se encuentran seguros, asignando categorías de permisos
    • Integridad: Recuperación de datos hasta un estado consistente.
    • Respaldo: Copias de respaldo de los datos y su restauración a partir de ellas.
    • Control de concurrencia: Controlar los accesos simultáneos a la base de datos.
    • Manejo de transacciones: Mecanismos para programar las modificaciones de los datos.
    • Tiempo de respuesta: Minimizar el tiempo de almacenamiento de cambios y de la espera de la información solicitada.[4]

  • Ejemplifica y justifica en que proyecto de los que has realizado hubiera sido conveniente utilizar una base de datos.
    Para el proyecto de Diseño Interactivo, ya que al ser un sitio web sobre el arte en el cine contaba con algunas secciones en las que los usuarios tenían la oportunidad de dejar algunas de sus críticas sobre las películas, así como también de sus recomendaciones, contando con una base de datos ya recolectados los datos de los usuarios (nombre, edad, correo, aportacion) se podría haber actualizado el contenido del sitio, así como tambíen hacer la notificación al resto de los usuarios sobre el nuevo material publicado.
