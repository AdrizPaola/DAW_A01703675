¿Por qué es una buena práctica separar el controlador de la vista?

Porque de ese modo se pueden separar los componentes de nuestra aplicación en base a la responsabilidad que tienen, por lo que, cuando se realiza un cambio en alguna parte del código este no afectará ninguna otra parte. [1]

Aparte de los arreglos $_POST y $_GET, ¿qué otros arreglos están predefinidos en php y cuál es su función?

  1. $_GLOBALS: Hace referencia a todas las variables disponibles en el ámbito global.
  2. $_SERVER: Es un array que contiene información, tales como cabeceras, rutas y ubicaciones de script.
  3. $_FILES: Una variable tipo array asociativo de variables pasadas al script actual a través de Cookies HTTP.
  4. $_COOKIE: es un array que contiene información, tales como cabeceras, rutas y ubicaciones de script.
  5. $_REQUEST: Un array asociativo que por defecto contiene el contenido de $_GET, $_POST y $_COOKIE.
  6. $_ENV: Una variable tipo array asociativo de variables pasadas al script actual a través del método del entorno. [2]

¿Qué es XSS y cómo se puede prevenir?

Es una vulnerabilidad conocida como Cross-site Scripting en la que los atacantes explotan la confianza que un usuario tiene en un sitio en particular y volver peligroso para sus usuarios a un sitio legítimo. Para evitarlo se puede contar con una solución de seguridad instalada y actualizada. Los usuarios lo pueden prevenir viendo el enlace URL y asegurándose que sea seguro. Los desarrolladores deben prevenir los ataques XSS al escapar de su output usando htmlspecialchars () o htmlentities (). Ambas funciones de PHP convierten los caracteres problemáticos en entidades HTML, lo que hace que el código inyectado se imprima de manera inofensiva y no se renderice. [4]
