Preguntas:

¿Qué importancia tiene AJAX en el desarrollo de RIA's (Rich Internet Applications)?

La tecnología RIA tiene como objetivo combinar las ventajas de las aplicaciones de escritorio con las de las aplicaciones web, enriqueciendo al cliente. Los productos JavaScript / AJAX son apropiados en situaciones donde el usuario espera que una aplicación funcione incondicionalmente en los navegadores. Al comprar un DVD o reservar un vuelo, por ejemplo, nadie querrá instalar un complemento. [1]

¿Qué implicaciones de seguridad tiene AJAX? ¿Dónde se deben hacer las validaciones de seguridad, del lado del cliente o del lado del servidor?

Además de las implicaciones de SQL Inyection y Cross Side Scripting se agregan algunas consideraciones que se mencionan a continuación: Las validaciones de seguridad de deben de hacer, como siempre, tanto del lado del cliente como del lado del servidor.

Del lado del cliente (Javascript) se dan las siguientes recomendaciones:

  • Usar .innerText en lugar de .innerHtml
  • No usar eval()
  • No depender de la lógica de negocio del cliente.
  • Evitar usar XML o JSON dinámicamente.
  • No encriptar del lado del cliente.

Del lado del servidor se dan las siguientes recomendaciones:

  • Usar protección CSRF (Cross-Site Request Forgery)
  • Protegesre ante JSON Hijacking para buscadores anteriores.
  • Evitar escribit código de searializacion. [2]

¿Qué es JSON?

Javascript Object Notation es un formato de texto para serializar datos estructurados (cadenas, números, booleanos, nulos, objetos y arreglos). Proviene del lenguaje de programación ECMAScript. [3]
