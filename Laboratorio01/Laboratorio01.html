Investigación

  • URL: Se le considera así a la ubicación de un archivo o una página web en Internet.[1]

  • MIME: (Extensiones Multipropósito de Correo de Internet), es un estándar que permite expandir capacidades del correo electrónico como el anexo de imágenes.[2]

  • Métodos HTTP: Métodos de petición para señalar acciones sobre un recurso determinado en el servidor.[3]

  • Códigos de errores HTTP: Son los códigos de estado en las respuestas HTTP que el servidor devuelve a una petición.[4]

  • XML: Lenguaje de Marcado Extensible, similar a HTML pero con la diferencia de que XML separa el contenido de la presentación. Facilita la representación y transmisión de información y sirve como marco para otros lenguajes.[7]

  • XHTML: Versión de HTML basada en XML. Tiene como fin que sea utilizado como lenguaje de contenidos pero que sea conforme a XML y funcione en agentes de usuario HTML 4.0.[8]

  • HTML5: Última versión de HTML que gracias a sus nuevos elementos permite a las aplicaciones ser de gran alcance.[9]

  • JS: JavaScript. Es el lenguaje de programación que permite agregar interactividad a los sitios web mejorando así la experiencia de usuario de los visitantes. Es práctico, útil y está disponible en cualquier navegador web. [10]

  • PHP: Hypertext Pre-Processor. Es un lenguaje de programación que se utiliza principalmente para crear páginas web así como también para desarrollar contenido dinámico y trabajar con bases de datos y HTML. [11]

  • JSP: JavaServer Pages, permite incluir código Java en el desarrollo sitios web, por lo cual se puede generar contenido dinámico del lado del servidor. [12]

  • AJAX: Asynchronous JavaScript and XML. Permite que las aplicaciones web funciones de una manera asíncrona por lo que que se procesa cualquier solicitud al servidor en segundo plano. De este modo cualquier aplicación web puede enviar y recuperar datos del servidor sin tener que volver a cargar toda la página.[13]

Preguntas

  • ¿Cuál es la diferencia entre Internet y la World Wide Web? Internet es una red de redes que permite la conexión entre dispositivos que utilizan el mismo protocolo TCP/IP, mientras que la WWW es un servicio que pertenece a la red de Internet y nos permite visualizar sitios web. [14]

  • ¿Cuál es el propósito de los métodos HTTP: GET, HEAD, POST, PUT, PATCH, DELETE? Son un conjunto de métodos de petición que sirven para indicar acciones a realizar para cierto recurso.
    1. GET: para recuperar información en forma de entidad (únicamente datos)
    2. HEAD: para recuperar información pero solamente líneas y headers sin el body de la respuesta del servidor.
    3. POST: para enviar una entidad a un recurso, lo que causa un cambio en el estado del servidor
    4. PUT: para remplazar un recurso
    5. PATCH: para remplazar parcialmente a un recurso
    6. DELETE: para borrar un recurso [15]

  • ¿Qué método HTTP se debe utilizar al enviar un formulario HTML, por ejemplo cuando ingresas tu usuario y contraseña en algún sitio? ¿Por qué? Se debe utilizar el método POST ya que así los datos no serán visibles al usuario de la web.[15]

  • ¿Qué método HTTP se utiliza cuando a través de un navegador web se accede a una página a través de un URL? Se utiliza el método GET, ya que así se obtiene el recurso de la página web solicitada. Los datos son visibles por la URL.[15]

  • Un servidor web devuelve una respuesta HTTP con código 200. ¿Qué significa esto? ¿Ocurrió algún error? Indica que la solicitud se ha realizado con éxito, sin ningún error.[16]

  • ¿Es responsabilidad del desarrollador corregir un sitio web si un usuario reporta que intentó acceder al sitio y se encontró con un error 404? ¿Por qué? Cuando se ha garantizado al cliente/usuario que el recurso existe y puede ser consultado entonces si debe de trabajarse en la corección del error, pero en caso de ser por una búsqueda de algún recurso inexistente entonces no hay nada que arreglar.

  • ¿Es responsabilidad del desarrollador corregir un sitio web si un usuario reporta que intentó acceder al sitio y se encontró con un error 500? ¿Por qué? Al ser un error del lado del servidor del sitio web y no de un fallo con el ordenador, claro que es trabajo del desarrollador solucionar el problema.[18]

  • ¿Qué significa que un atributo HTML5 esté depreciado o desaprobado (deprecated)? Menciona algunos elementos de HTML 4 que en HTML5 estén desaprobados. Significa que no son recomendados para utilizarse en las nuevas versiones de los lenguajes. Ej: acronym, big, center, frame, frameset, noframes, tt. [17]

  • ¿Cuáles son las diferencias principales entre HTML 4 y HTML5? HTML5 convierte a los navegadores en plataformas de aplicaciones gracias a JavaScript, CSS y HTML, en esta nueva versión se agregan nuevos atributos y elementos como header, section, article, audio, video, etc., lo cual permite que los sitios sean más sencillos de leer y semánticos.

  • ¿Qué componentes de estructura y estilo tiene una tabla? Para insertar la tabla se utiliza la etiqueta table y para definir las filas tr. Cada fila contiene celdas que se definen con td o th (celdas de encabezados). Si se quiere añadir un título a la tabla se utiliza caption. Para dividir la tabla en las secciones: encabezado, cuerpo y pie se implementan las etiquetas thead, tbody, tfoot, respectivamente. Existen algunos atributos como colspan y rowspan y estilos como border, border-spacing, border-collapse, padding, margin, etc.[19]

  • ¿Cuáles son los principales controles de una forma HTML5? Input con una gran variedad de tipos como number, range, email, date, time, color, search y elementos como datalist.[17]

  • ¿Qué tanto soporte HTML5 tiene el navegador que utilizas? Puedes utilizar la siguiente página para descubrirlo: http://html5test.com/ La puntuación obtenida fue de 491 de 555 puntos, así que pues es un nivel regular alto para el navegador que utilizo que es Firefox 68.0 en Windows 10. [20]

  • Sobre el ciclo de vida y desarrollo de los sistemas de información:
    ¿Cuál es el ciclo de vida de los sistemas de información?
    1. Investigación preliminar: Investigación de los procesos de la organización para la identificación de necesidades.
    2. Análisis detallado de la situación: Comprender críticamente la gestión de la información en la entidad.
    3. Diseño del nuevo sistema: Diseñar los elementos fundamentales del nuevo sistema.
    4. Desarrollo y documentación: Mantener la documentación conforme al avance del desarrollo del sotware.
    5. Implantación: Introducción en la actividad práctica.
    6. Operaciones y Mantenimiento: Adecuar el sistema a futuros cambios en el entorno. [5][6]


    ¿Cuál es el ciclo de desarrollo de sistemas de información?
    1. Obtener Requesitos
    2. Análisis del sistema
    3. Diseño del sistema
    4. Construcción de prototipo
    5. Evaluación/Validación. [21][22]
