Preguntas:

¿Por qué es una buena práctica separar el modelo del controlador?

Porque facilita la detección de errores ya que nos permite dividir en partes el código, y así siendo independientes si se realiza algún cambio no se afectaría el resto. De este modo se mantiene el código lo más simple posible ya que cada parte en específico tiene su propia función. [1]

¿Qué es SQL Injection?

Es una vulnerabilidad en la que el atacante envía instrucciones SQL por medio de las entradas o formas de la página para que entre dentro de un código SQL con fines de manipular una base de datos. Y en consecuencia administrar la base de datos.[2]

¿Qué técnicas puedes utilizar para evitar ataques de SQL Injection?

Verificar los datos que se están recibiendo como entrada y comprobar que es el tipo de dato esperado. También se recomienda no brindar detalles extras de los errores que se generan.
