Preguntas

  • ¿Por qué es una buena práctica usar JavaScript para checar que sean válidos los inputs de las formas antes de enviar los datos al servidor?
    Porque así se puede notificar al usuario que se ha cometido algún error sin la necesidad de esperar la respuesta del servidor. Al notificar este tipo de errores de manera inmediata se mejora la experiencia del usuario. Cuando se validan formularios se comprueba que los datos ingresados cumplan con los requisitos para ser aceptados, de modo que se previene el envío de formularios erróneos al servidor que se tenga que esperar una respuesta que al final de igual manera será errónea. [1]


  • ¿Cómo puedes saltarte la seguridad de validaciones hechas con JavaScript?
    Se puede hacer simplemente desactivando JavaScript del navegador. Para ello se abre el navegador, la configuración, se hace clic en Configuracion avanzada, después en "Privacidad y Seguridad" se selecciona Configuración de contenido, clic en JavaScript y Actve (no permitido). [2]


  • Si te puedes saltar la seguridad de las validaciones de JavaScript, entonces ¿por qué la primera pregunta dice que es una buena práctica?
    ´Porque para muchas personas esto puede no tener importancia, pero pues en realidad el cliente obtiene beneficios ya que obtiene respuestas inmediatas, además de que de este modo se previenen problemas de seguridad.
