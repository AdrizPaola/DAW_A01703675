Preguntas:

¿Por qué es importante hacer un session_unset() y luego un session_destroy()?

Porque con session_unset() se pueden borrar variables específicas de una sesión lo cual nos permite alterar la información de una sesión, mientras que con un session_destroy() elimina todo lo que se almacena de la sesión actual, generalmente se usa cuando el usuario va a ser desconectado.[1]

¿Cuál es la diferencia entre una variable de sesión y una cookie?

La principal diferencia entre una sesión y una cookie es que los datos de la sesión se almacenan en el servidor, mientras que las cookies almacenan los datos en el navegador del visitante. Las sesiones usan un identificador de sesión para ubicar los datos de sesión de un usuario en particular. Este identificador de sesión normalmente se almacena en el navegador web del usuario en una cookie, pero los datos confidenciales que deben ser más seguros, como la identificación del usuario, el nombre, etc., siempre permanecerán en el servidor. [2]

¿Qué técnicas se utilizan en sitios como facebook para que el usuario no sobreescriba sus fotos en el sistema de archivos cuando sube una foto con el mismo nombre?

Se revisan las variables de sesión y en caso de que ya existan otras con el mismo nombre se le notifica al usuario.

¿Qué es CSRF y cómo puede prevenirse?

Se producen cuando un atacante provoca que el usuario ejecute aaciones de forma no intencionada en algún sitio donde se había uniciado sesión. Tolo que puede hacer el uduario como actualizar el perfil o postear mensajes lo puede hacer también el atacante. Para prevenirlo se debe asociar al usuario con un tipo de identificador único (token) que se puede estar verificando después.
